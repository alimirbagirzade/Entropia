# ADR 0001 — Naming conventions (GAP-21)

- **Date:** 2026-07-13
- **Status:** Accepted
- **Supersedes / relates to:** Master Technical Reference §11 — *"Endpoint isimleri
  örnektir; proje genelinde REST kaynak isimlendirmesi benimsenecektir."* This ADR is
  that adoption: the convention the codebase already follows is now the written,
  enforced standard.

## Context

By V1 completion the project had grown to 158 HTTP endpoints, 97 database tables, ~196
error codes, a 24-page frontend, and a large Python domain. The naming was in practice
already remarkably consistent, but the convention lived only in the collective habit of
the code — it was never written down as the authoritative standard, and the master
reference explicitly deferred it ("benimsenecektir" — *will be adopted*). GAP-21 closes
that gap: it records the adopted conventions and adds an automated guard
(`backend/tests/unit/test_naming_conventions.py`) so future drift is caught in CI rather
than by reviewer memory.

A single genuine drift existed at the time of writing — the endpoint literal segment
`/future-dev/graphic_view/overview` was snake_case while every other segment was
kebab-case. It was normalized to `/future-dev/graphic-view/overview` (backend route,
query docstring, frontend fetch + test stub, regenerated `docs/openapi.json`) in the
same slice. The `graphic_view` **capability key** value (a stable domain identifier in
seed data, DB rows, and the agent tool gateway) is deliberately left unchanged — it is a
registry key, not a REST resource segment.

## Decision

### HTTP / REST

- **Base prefix:** `settings.api_base_path` (`/api/v1`). Versioned in the path.
- **Resource segments:** lowercase **kebab-case, plural** — `/market-datasets`,
  `/rationale-families`, `/embedded-system-packages`, `/trading-signals`.
- **Path parameters:** **snake_case**, named `<noun>_id` — `{entity_id}`, `{root_id}`,
  `{request_id}`, `{workspace_id}`. Registry-key parameters carry the key verbatim
  (e.g. `/capabilities/{capability_key}` where the value is a snake_case key).
- **Sub-actions on a resource:** kebab-case path suffixes — `/{id}/approve`,
  `/{id}/deprecate`, `/{id}/generate-candidate`, `/{id}/schema-mapping`.
- **Custom methods** (non-CRUD verbs that don't fit a sub-resource): the Google-AIP
  colon form `resource:verb`, verb in kebab-case — `:batch`, `:restore`, `:upload`.
- **Command namespaces:** a kebab-case prefix that groups a command lifecycle —
  `/create-package/requests/...`.
- **Singleton / operational endpoints:** short kebab-case nouns — `/me`, `/live`,
  `/ready`, `/metrics`, `/login`, `/logout`, `/signup`, `/bootstrap-status`,
  `/mainboards/default`.
- **Query parameters:** snake_case — `state`, `cursor`, `limit`, `q`, `correlation_id`,
  `actor_type`.
- **Authoritative contract:** `docs/openapi.json` (regenerated by `make openapi`, drift
  is a CI failure via `make openapi-check`). Endpoint spellings quoted inside `docs/spec`
  and historical `docs/STAGE*` handoffs are illustrative examples (master ref §11) and are
  not chased when a path changes — the OpenAPI snapshot is the source of truth.

### Concurrency / idempotency tokens

- **Body-form OCC tokens:** `expected_<subject>` — `expected_row_version`,
  `expected_head_revision_id`, `expected_draft_row_version`, `expected_fingerprint`,
  `expected_registry_version`, `expected_request_version`. When both a body token and an
  `If-Match` header are present, the **body wins** (per-route contract).
- **Header-form OCC tokens:** `If-Match` (opaque `"rv-N"`), `X-Registry-Version`,
  `X-Request-Version`.
- **Idempotency:** the `Idempotency-Key` request header on state-changing POSTs.

### Database (PostgreSQL / SQLAlchemy / Alembic)

- **Tables:** **snake_case, singular** — `market_dataset`, `backtest_result`,
  `capability_activation_event`. (Project convention is singular table names; it is
  applied uniformly.)
- **Columns:** snake_case.
- **Primary key:** the semantic form `<entity>_id` — `capability_id`, `event_id`,
  `view_dataset_id` (not a generic `id`).
- **Foreign key columns:** `<referenced_entity>_id`.
- **Indexes:** `ix_<table>_<descriptor>` — `ix_audit_events_event_kind_trgm`.
- **Migrations:** `NNNN_<slug>.py` under `alembic/versions/`, zero-padded sequential —
  `0028_resource_share.py`.

### Identifiers / registry keys / error codes

- **Registry & enum key values** (capability keys, resolver keys, taxonomy tokens):
  snake_case string literals — `graphic_view`, `view_dataset`. These are stable domain
  identifiers; renaming one is a data migration, not a cosmetic change.
- **Error codes** (`AppError.code`): **UPPER_SNAKE_CASE** — `VALIDATION_ERROR`,
  `STALE_REVISION`, `CAPABILITY_NOT_ACTIVE`.

### Python

- Modules & functions: `snake_case`. Classes / types: `PascalCase`. Constants:
  `UPPER_SNAKE_CASE`. One exported concept per module; module-private symbols prefixed
  with a single underscore.

### Frontend (TypeScript / React)

- **`lib/` modules and their functions/hooks:** camelCase files, camelCase exports —
  `capability.ts`, `useGraphicViewOverview`.
- **Page/components:** PascalCase files and components — `FutureDev.tsx`.
- **Wire fields** (JSON to/from the backend): snake_case, mirroring the server contract
  verbatim — the client never re-cases server fields.
- **react-query keys:** array form under a stable domain prefix — `["capabilities", ...]`,
  `["market-data", ...]` — so an invalidation of the prefix sweeps every derived key.

## Consequences

- The convention is now **written and machine-enforced**. `test_naming_conventions.py`
  introspects the live app (OpenAPI paths), `Base.metadata` (tables), and the `AppError`
  hierarchy (codes); a new endpoint/table/error code that breaks the convention fails a
  unit test, not just review.
- The guard checks only the objectively introspectable, contract-bearing surfaces
  (REST, DB, error codes). Frontend and Python-internal naming stay convention + review
  enforced (documented here, not asserted by this test) to avoid brittle source scanning.
- Endpoint renames remain a coordinated backend + frontend + OpenAPI change. Because the
  frontend and backend ship from one repo, such a rename is safe in practice, but it is
  still a contract change and is treated as one.
- New ADRs are added as `docs/adr/NNNN-<slug>.md` and indexed in `docs/adr/README.md`.
